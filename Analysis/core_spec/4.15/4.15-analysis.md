# TCP Communication FSM Analysis

## Behavioral Elements from Specification

### 1. Transport Protocol Selection
- **Constraint**: Node using TCP SHALL NOT use MRP reliability semantics
- **Choice**: Nodes MAY choose TCP as transport
- **State implication**: Transport selection affects reliability mechanism

### 2. Secure Session Establishment (Section 4.15.1)
- **Choice**: Pair of TCP-supporting nodes MAY choose TCP for session
- **Typical pattern**: Node SHOULD have either MRP or TCP session with peer (not both)
- **Exception**: MAY have both for specific use cases (e.g., OTA)
- **Session lifetime**: MAY be long-lived or short-lived
- **Critical constraint**: Session unusable when connection broken/closed
- **Session removal**: Nodes MAY remove session when connection goes down
- **Session retention**: If retained, SHALL be marked for reconnection requirement
- **Resumption**: Session resumption state MAY be retained for expedited re-establishment

### 3. TCP Connection Configuration (Section 4.15.2.1)

#### Configuration Parameters:
1. **Connection Establishment Timeout**
   - MAY configure timeout for connection attempt
   - Action: Notify application on timeout

2. **TCP Keep Alive**
   - MAY configure for long-lived connections
   - Parameters (SHALL configure):
     - TCP_KEEP_ALIVE_TIME: interval before first probe
     - TCP_KEEP_ALIVE_INTERVAL: interval between probes
     - TCP_KEEP_ALIVE_PROBES: number of probes before declaring dead
   - Formula: Total_Timeout = TIME + (INTERVAL × PROBES)

3. **TCP User Timeout**
   - MAY configure timeout for unacknowledged data
   - Action: Force close connection on expiry

### 4. TCP Connection Closures (Section 4.15.2.2)

#### Closure Triggers:
- Upper layer instruction
- Keep Alive Timeout expiry on idle connection
- TCP User Timeout expiry (unacknowledged data)

#### Closure Protocol:
- **Peer closes**: Node SHALL close its end AND notify application
- **Exchange closure**: Active exchanges SHOULD be closed (unusable on closed connection)

#### Re-establishment:
- Either side MAY re-establish
- SHOULD back-off (Fibonacci sequence) random time before retry
- On incoming connection request: SHOULD discard own backoff timer to same peer
- Goal: Prevent connection races (works for "most scenarios")
- SHOULD reap old unused connections

### 5. Message Size Management (Section 4.15.2.3)

#### Maximum Message Size:
- Platform MAY configure Maximum Message Size
- **Oversized message**: If received header shows size > max:
  - SHALL close connection
  - SHOULD send STATUS_REPORT with MESSAGE_TOO_LARGE before close

#### Dynamic Configuration:
- Node using both MRP and TCP SHOULD dynamically adjust max size based on current Exchange transport

## State Variables Identification

### Connection State Variables:
- `connection_status`: {not_established, establishing, established, idle, peer_closed, closing, closed}
- `establishment_timer`: {active, inactive, expired}
- `keep_alive_time_remaining`: integer (seconds)
- `keep_alive_probe_count`: integer (0 to TCP_KEEP_ALIVE_PROBES)
- `user_timeout_remaining`: integer (seconds)
- `unacknowledged_data_pending`: boolean
- `backoff_timer_active`: boolean
- `backoff_duration`: integer (Fibonacci sequence value)
- `last_activity_time`: timestamp

### Session State Variables:
- `session_status`: {not_established, active, retained_disconnected, removed}
- `session_marked_for_reconnect`: boolean
- `resumption_state_retained`: boolean
- `transport_type`: {TCP, MRP}
- `mrp_reliability_enabled`: boolean

### Exchange State Variables:
- `active_exchanges`: set of exchange_ids
- `exchange_connection_mapping`: map(exchange_id → connection_id)

### Message Size Variables:
- `current_max_message_size`: integer (bytes)
- `default_tcp_max_size`: integer
- `default_mrp_max_size`: integer (1280 bytes for IPv6 MTU)
- `received_message_size`: integer

### Configuration Variables:
- `tcp_keep_alive_time`: integer (configured)
- `tcp_keep_alive_interval`: integer (configured)
- `tcp_keep_alive_probes`: integer (configured)
- `tcp_user_timeout`: integer (configured)
- `establishment_timeout`: integer (configured)

## State Identification

### S1: Initial (No Connection)
- connection_status = not_established
- session_status = not_established
- No timers active

### S2: Connection Establishing
- connection_status = establishing
- establishment_timer = active
- session_status = not_established

### S3: Connection Established, Session Active
- connection_status = established
- session_status = active
- transport_type = TCP
- mrp_reliability_enabled = false
- active_exchanges may be non-empty

### S4: Connection Idle (Keep Alive Active)
- connection_status = idle
- session_status = active
- keep_alive_time_remaining > 0
- No data activity

### S5: Keep Alive Probing
- connection_status = idle
- keep_alive_time_remaining = 0
- keep_alive_probe_count < TCP_KEEP_ALIVE_PROBES
- Sending keep-alive probes

### S6: Peer Closed Connection
- connection_status = peer_closed
- session_status = active OR retained_disconnected
- Must close local end

### S7: Connection Closing (Local Initiated)
- connection_status = closing
- Reason: upper_layer_command OR keep_alive_timeout OR user_timeout
- active_exchanges being notified

### S8: Connection Closed, Session Removed
- connection_status = closed
- session_status = removed
- All exchanges closed

### S9: Connection Closed, Session Retained
- connection_status = closed
- session_status = retained_disconnected
- session_marked_for_reconnect = true
- resumption_state_retained = true/false

### S10: Backoff Before Reconnection
- connection_status = closed
- backoff_timer_active = true
- backoff_duration = Fibonacci(attempt_number) + random

### S11: Reconnection Attempt
- connection_status = establishing
- establishment_timer = active
- session_status = retained_disconnected (if resuming)

### S12: Data Transmission Active
- connection_status = established
- unacknowledged_data_pending = true
- user_timeout_remaining > 0

### S13: Unacknowledged Data Timeout
- connection_status = established
- unacknowledged_data_pending = true
- user_timeout_remaining = 0
- Must force close

### S14: Oversized Message Received
- connection_status = established
- received_message_size > current_max_message_size
- Must close and send error

### S15: Establishment Timeout
- connection_status = establishing
- establishment_timer = expired
- Must notify application and fail

### S16: Dual Transport (MRP + TCP Sessions)
- Two parallel sessions to same peer
- One MRP session (not modeled here)
- One TCP session (modeled)
- Justified use case (e.g., OTA)

## Transition Identification

### T1: Initiate Connection
- **From**: S1 (Initial)
- **To**: S2 (Establishing)
- **Trigger**: initiate_tcp_connection(peer)
- **Guard**: supports_tcp(self) && supports_tcp(peer)
- **Actions**:
  - connection_status := establishing
  - establishment_timer := establishment_timeout
  - send_syn_packet(peer)

### T2: Connection Established Successfully
- **From**: S2 (Establishing)
- **To**: S3 (Active)
- **Trigger**: connection_established_event
- **Guard**: establishment_timer > 0
- **Actions**:
  - connection_status := established
  - session_status := active
  - transport_type := TCP
  - mrp_reliability_enabled := false
  - establishment_timer := inactive
  - notify_application(connection_success)

### T3: Establishment Timeout
- **From**: S2 (Establishing)
- **To**: S15 (Establishment Failed)
- **Trigger**: timer_tick
- **Guard**: establishment_timer == 0
- **Actions**:
  - connection_status := not_established
  - notify_application(connection_timeout)
  - generate_event(establishment_failure)

### T4: Connection Active to Idle (No Activity)
- **From**: S3 (Active)
- **To**: S4 (Idle)
- **Trigger**: no_data_activity_detected
- **Guard**: time_since_last_data >= idle_threshold
- **Actions**:
  - connection_status := idle
  - keep_alive_time_remaining := tcp_keep_alive_time
  - last_activity_time := current_time

### T5: Keep Alive Time Expires
- **From**: S4 (Idle)
- **To**: S5 (Probing)
- **Trigger**: timer_tick
- **Guard**: keep_alive_time_remaining == 0
- **Actions**:
  - keep_alive_probe_count := 0
  - send_keep_alive_probe()
  - keep_alive_time_remaining := tcp_keep_alive_interval

### T6: Keep Alive Probe Acknowledged
- **From**: S5 (Probing)
- **To**: S3 (Active)
- **Trigger**: keep_alive_ack_received
- **Guard**: keep_alive_probe_count < tcp_keep_alive_probes
- **Actions**:
  - connection_status := established
  - keep_alive_probe_count := 0
  - keep_alive_time_remaining := tcp_keep_alive_time

### T7: Keep Alive Probe No Response
- **From**: S5 (Probing)
- **To**: S5 (Probing)
- **Trigger**: timer_tick
- **Guard**: keep_alive_time_remaining == 0 && keep_alive_probe_count < tcp_keep_alive_probes
- **Actions**:
  - keep_alive_probe_count := keep_alive_probe_count + 1
  - send_keep_alive_probe()
  - keep_alive_time_remaining := tcp_keep_alive_interval

### T8: Keep Alive Total Timeout (Connection Dead)
- **From**: S5 (Probing)
- **To**: S7 (Closing)
- **Trigger**: timer_tick
- **Guard**: keep_alive_probe_count >= tcp_keep_alive_probes
- **Actions**:
  - connection_status := closing
  - notify_application(connection_dead)
  - generate_event(keep_alive_timeout)
  - close_reason := keep_alive_timeout

### T9: Data Sent (Start User Timeout)
- **From**: S3 (Active)
- **To**: S12 (Data Pending)
- **Trigger**: send_data(data)
- **Guard**: true
- **Actions**:
  - unacknowledged_data_pending := true
  - user_timeout_remaining := tcp_user_timeout
  - transmit_data(data)

### T10: Data Acknowledged
- **From**: S12 (Data Pending)
- **To**: S3 (Active)
- **Trigger**: data_ack_received
- **Guard**: true
- **Actions**:
  - unacknowledged_data_pending := false
  - user_timeout_remaining := 0

### T11: User Timeout Expires
- **From**: S12 (Data Pending)
- **To**: S13 (User Timeout)
- **Trigger**: timer_tick
- **Guard**: user_timeout_remaining == 0 && unacknowledged_data_pending == true
- **Actions**:
  - connection_status := closing
  - notify_application(data_timeout)
  - generate_event(user_timeout_expired)

### T12: Force Close After User Timeout
- **From**: S13 (User Timeout)
- **To**: S7 (Closing)
- **Trigger**: force_close_command
- **Guard**: true
- **Actions**:
  - close_reason := user_timeout
  - force_close_connection()

### T13: Upper Layer Requests Close
- **From**: S3 (Active) OR S4 (Idle) OR S12 (Data Pending)
- **To**: S7 (Closing)
- **Trigger**: upper_layer_close_request
- **Guard**: true
- **Actions**:
  - connection_status := closing
  - close_reason := upper_layer_command
  - initiate_graceful_close()

### T14: Peer Closes Connection
- **From**: S3 (Active) OR S4 (Idle) OR S5 (Probing) OR S12 (Data Pending)
- **To**: S6 (Peer Closed)
- **Trigger**: peer_close_notification
- **Guard**: true
- **Actions**:
  - connection_status := peer_closed
  - notify_application(peer_closed_connection)

### T15: Local Close After Peer Close
- **From**: S6 (Peer Closed)
- **To**: S7 (Closing)
- **Trigger**: close_local_end_command
- **Guard**: true
- **Actions**:
  - close_local_connection()
  - close_reason := peer_initiated

### T16: Close All Exchanges
- **From**: S7 (Closing)
- **To**: S8 (Closed, Session Removed) OR S9 (Closed, Session Retained)
- **Trigger**: connection_closed_event
- **Guard**: true (branches based on policy)
- **Actions**:
  - connection_status := closed
  - close_all_active_exchanges()
  
### T16a: Remove Session
- **From**: S7 (Closing)
- **To**: S8 (Closed, Session Removed)
- **Trigger**: connection_closed_event
- **Guard**: session_retention_policy == remove
- **Actions**:
  - connection_status := closed
  - session_status := removed
  - close_all_active_exchanges()
  - clear_session_state()

### T16b: Retain Session
- **From**: S7 (Closing)
- **To**: S9 (Closed, Session Retained)
- **Trigger**: connection_closed_event
- **Guard**: session_retention_policy == retain
- **Actions**:
  - connection_status := closed
  - session_status := retained_disconnected
  - session_marked_for_reconnect := true
  - may_retain_resumption_state()
  - close_all_active_exchanges()

### T17: Start Backoff Timer
- **From**: S8 (Closed, Removed) OR S9 (Closed, Retained)
- **To**: S10 (Backoff)
- **Trigger**: reconnection_desired_event
- **Guard**: reconnection_allowed == true
- **Actions**:
  - backoff_duration := calculate_fibonacci_backoff(attempt_number)
  - backoff_timer_active := true
  - reconnection_attempt_number := reconnection_attempt_number + 1

### T18: Backoff Timer Expires
- **From**: S10 (Backoff)
- **To**: S11 (Reconnecting)
- **Trigger**: timer_tick
- **Guard**: backoff_duration == 0
- **Actions**:
  - backoff_timer_active := false
  - connection_status := establishing
  - establishment_timer := establishment_timeout
  - initiate_reconnection(peer)

### T19: Incoming Connection During Backoff
- **From**: S10 (Backoff)
- **To**: S2 (Establishing)
- **Trigger**: incoming_connection_request(peer)
- **Guard**: backoff_timer_active == true
- **Actions**:
  - backoff_timer_active := false
  - backoff_duration := 0
  - connection_status := establishing
  - accept_incoming_connection(peer)

### T20: Reconnection Successful
- **From**: S11 (Reconnecting)
- **To**: S3 (Active)
- **Trigger**: connection_established_event
- **Guard**: session_status == retained_disconnected
- **Actions**:
  - connection_status := established
  - session_status := active
  - session_marked_for_reconnect := false
  - may_use_resumption_state()
  - notify_application(reconnection_success)

### T21: Oversized Message Header Received
- **From**: S3 (Active) OR S12 (Data Pending)
- **To**: S14 (Oversized Message Error)
- **Trigger**: receive_message_header(header)
- **Guard**: extract_message_size(header) > current_max_message_size
- **Actions**:
  - received_message_size := extract_message_size(header)
  - generate_event(oversized_message_detected)

### T22: Send Error and Close After Oversized Message
- **From**: S14 (Oversized Message Error)
- **To**: S7 (Closing)
- **Trigger**: handle_oversized_message
- **Guard**: true
- **Actions**:
  - send_status_report(MESSAGE_TOO_LARGE)
  - close_reason := message_too_large
  - initiate_close()

### T23: Dynamic Max Size Adjustment (MRP Exchange)
- **From**: S3 (Active)
- **To**: S3 (Active)
- **Trigger**: exchange_started(exchange_id, transport)
- **Guard**: transport == MRP
- **Actions**:
  - current_max_message_size := default_mrp_max_size

### T24: Dynamic Max Size Adjustment (TCP Exchange)
- **From**: S3 (Active)
- **To**: S3 (Active)
- **Trigger**: exchange_started(exchange_id, transport)
- **Guard**: transport == TCP
- **Actions**:
  - current_max_message_size := default_tcp_max_size

### T25: Reap Idle Connection
- **From**: S4 (Idle)
- **To**: S7 (Closing)
- **Trigger**: timer_tick
- **Guard**: time_since_last_activity > reap_threshold
- **Actions**:
  - connection_status := closing
  - close_reason := resource_conservation
  - initiate_graceful_close()

## Function Definitions

### F1: calculate_fibonacci_backoff(attempt_number)
- **Parameters**: attempt_number (integer, 1-based)
- **Returns**: backoff_duration (integer, milliseconds)
- **Description**: Computes backoff duration using Fibonacci sequence with random jitter
- **Algorithm**: fibonacci(attempt_number) * base_interval + random(0, jitter_range)
- **Usage**: T17 when entering backoff state

### F2: send_keep_alive_probe()
- **Parameters**: None
- **Returns**: void
- **Description**: Sends TCP keep-alive packet to peer
- **Algorithm**: TCP layer probe mechanism (implementation-specific)
- **Usage**: T5, T7 during keep-alive probing

### F3: close_all_active_exchanges()
- **Parameters**: None
- **Returns**: void
- **Description**: Closes all Exchange objects tied to this connection
- **Algorithm**: Iterate active_exchanges set, call close() on each, notify application
- **Usage**: T16a, T16b when connection closes

### F4: extract_message_size(header)
- **Parameters**: header (bitstring, message header bytes)
- **Returns**: message_size (integer, bytes)
- **Description**: Parses message header to extract total payload size field
- **Algorithm**: Read size field from header structure per Matter message format
- **Usage**: T21 when checking for oversized messages

### F5: send_status_report(error_code)
- **Parameters**: error_code (enum: MESSAGE_TOO_LARGE, etc.)
- **Returns**: void
- **Description**: Sends STATUS_REPORT message with specified error code to peer
- **Algorithm**: Construct STATUS_REPORT with error_code, attempt send before close
- **Usage**: T22 when oversized message detected

### F6: may_retain_resumption_state()
- **Parameters**: None
- **Returns**: void
- **Description**: Optionally stores session resumption state for future reconnection
- **Algorithm**: If policy allows, store session keys and resumption token securely
- **Usage**: T16b when retaining session after disconnect

### F7: may_use_resumption_state()
- **Parameters**: None
- **Returns**: void
- **Description**: Optionally uses stored resumption state to expedite session establishment
- **Algorithm**: If resumption state exists and valid, use it to skip full handshake
- **Usage**: T20 during reconnection with retained session

### F8: supports_tcp(node)
- **Parameters**: node (node identifier)
- **Returns**: boolean
- **Description**: Checks if node advertises TCP support in DNS-SD TXT record
- **Algorithm**: Query DNS-SD record for TXT key 'T', return true if present
- **Usage**: T1 before initiating TCP connection

### F9: initiate_reconnection(peer)
- **Parameters**: peer (node identifier)
- **Returns**: void
- **Description**: Starts TCP connection establishment after backoff period
- **Algorithm**: Send SYN to peer, start establishment timer
- **Usage**: T18 when backoff expires

### F10: accept_incoming_connection(peer)
- **Parameters**: peer (node identifier)
- **Returns**: void
- **Description**: Accepts incoming TCP connection from peer
- **Algorithm**: Complete TCP handshake (SYN-ACK), establish connection
- **Usage**: T19 when peer initiates during local backoff

## Security Properties

### SP1: MRP Reliability Exclusion
- **Type**: Correctness
- **Description**: MRP reliability MUST NOT be used over TCP
- **States**: S3, S12 (any active connection state)
- **Invariant**: mrp_reliability_enabled == false when transport_type == TCP

### SP2: Session Unusability on Broken Connection
- **Type**: Security
- **Description**: Session cannot be used when connection is broken
- **States**: S6, S7, S8, S9
- **Invariant**: session_status != active when connection_status in {peer_closed, closing, closed}

### SP3: Session Marking for Reconnection
- **Type**: Security
- **Description**: Retained sessions must be marked before reuse
- **States**: S9, S11
- **Invariant**: session_marked_for_reconnect == true when session_status == retained_disconnected

### SP4: Connection Closure Notification
- **Type**: Consistency
- **Description**: Application must be notified of peer-initiated closure
- **States**: S6
- **Transition**: T14 must include notify_application()

### SP5: Exchange Closure on Connection Close
- **Type**: Consistency
- **Description**: All exchanges must be closed when connection closes
- **States**: S7, S8, S9
- **Transition**: T16a, T16b must call close_all_active_exchanges()

### SP6: Backoff Timer Cancellation
- **Type**: Correctness
- **Description**: Backoff timer must be cancelled on incoming connection
- **States**: S10
- **Transition**: T19 must set backoff_timer_active := false

### SP7: Oversized Message Rejection
- **Type**: Security
- **Description**: Messages exceeding max size must trigger connection closure
- **States**: S14
- **Transitions**: T21, T22 enforce size limit

### SP8: Dynamic Size Configuration
- **Type**: Correctness
- **Description**: Max message size must match transport type
- **States**: S3
- **Transitions**: T23, T24 adjust size per transport

## Security Assumptions

### A1: TCP Reliability
- **Assumption**: TCP provides reliable, in-order delivery
- **Type**: Explicit
- **If violated**: MRP reliability exclusion becomes unsafe

### A2: DNS-SD Integrity
- **Assumption**: DNS-SD records for TCP capability are trusted
- **Type**: Implicit
- **If violated**: Downgrade attacks possible

### A3: Secure Storage for Resumption State
- **Assumption**: Resumption state stored with proper access control
- **Type**: Implicit
- **If violated**: Session hijacking via storage compromise

### A4: Clock Synchronization
- **Assumption**: Timers operate with sufficient accuracy
- **Type**: Implicit
- **If violated**: Keep-alive and timeout mechanisms unreliable

## Edge Cases and Error States

### E1: Simultaneous Bidirectional Connection Attempts
- Spec says backoff "SHOULD prevent races for most scenarios"
- Not fully specified: what if both peers simultaneously connect after canceling backoff?
- Result: Two connections possible
- Not modeled as error state (spec allows it)

### E2: Backoff Without Upper Bound
- Fibonacci backoff can grow indefinitely
- Potential: Permanent inability to reconnect
- Not modeled (spec doesn't bound it)

### E3: Resumption State Replay
- Spec doesn't mandate freshness checks
- Potential: Replay of old resumption data
- Not modeled (out of scope for this section)

### E4: Partial Keep-Alive Probe Sequence
- Formula assumes all probes fail
- Actual detection time varies if connection breaks mid-sequence
- Modeled as: Full probe sequence required
