# MCP Search Evaluation — hybrid_search vs search_documentation 🔍

## TL;DR ✅

- **Recommended primary tool:** **`hybrid_search`** — best for exploratory, context-rich queries and navigation across pages and diagrams.
- **Recommended secondary tool:** **`search_documentation`** — best for fast, focused snippet retrieval.
- Use **`get_page`** for exact page content (handles merged pages) and **`get_document_info`** for document-level stats.

## Why use this MCP vs VS Code default search (short) ⚖️

- **Semantic understanding:** MCP uses vector embeddings → supports natural language queries ("show me message framing over TCP"), not just literal text.
- **Context & navigation:** hybrid_search returns adjacent pages and semantically related pages (NEXT / PREV / related-discussing) using a Neo4j graph — VSCode search cannot provide those relationships.
- **Structured results:** MCP returns page metadata (page_range, is_merged, word_count) and can fetch a specific page (`get_page`) even if the PDF page was merged — VSCode search returns file/line hits only.
- **Diagrams & merged content:** MCP surfaces diagrams and sequence diagrams as searchable chunks; VSCode finds literal text only and can miss images/merged chunks.
- **Document-level ops:** `get_document_info` lists total chunks/merged chunks/relationships; handy for diagnostics and completeness checks (e.g., missing page footers). VSCode has no equivalent.

Caveats: MCP requires a running stack (Docker + Neo4j + Qdrant) and indexing step; VSCode search is instant for local files and remains useful for literal/regexp tasks.

## Recommended workflow ✨

- Exploration / spec reading: `hybrid_search(query, doc_filter='core_spec')` → follow `related_next` / `related_previous` → `get_page` to read full page.
- Quick lookup: `search_documentation(query, doc_filter='core_spec')` → take top chunk.
- Validation / diagnostics: `get_document_info('core_spec')` to check chunk counts and merged pages.

### Example MCP calls (pseudo):

```
hybrid_search("table of contents chapters list", doc_filter='core_spec')
search_documentation("message framing stream-oriented transports", doc_filter='core_spec')
get_page('core_spec', 125)
get_document_info('core_spec')
```
